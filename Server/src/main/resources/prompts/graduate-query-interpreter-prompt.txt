Core routing rules:
- An explicit tuition, fee, price, cost, or per-credit question always uses:
  resource=PROGRAM, operation=AGGREGATE,
  aggregation={"function":"AVG","field":"TUITION"}.
- Do not use PROGRAM/LIST or PROGRAM/DETAILS for an explicit tuition question.
- Use AGGREGATE only for a supported aggregation request.
- Use COMPARE only for an explicit objective comparison and include

- Preserve every explicit supported filter. Use null or [] when a filter is

- City-only location questions are valid and must not invent a university.
- University references are user-facing names or acronyms, never database IDs.

University extraction:
- Preserve the user reference in filters.universities, for example AUB, LAU,
  BAU, NDU, USJ, or a full university name. Java resolves acronyms, aliases,
  and canonical names against the authoritative catalog.
- Do not expand an acronym to a guessed name and do not invent a university
  from a faculty, department, city, or program.

Degree extraction:
- MASTER: master's, master, MSc, M.Sc., MS, M.S., MA, M.A., MBA, Master of
  Science, Master of Arts, Master of Business Administration, and equivalent
  master's wording.
- PHD: PhD, Ph.D., doctorate, doctoral degree, Doctor of Philosophy.
- DIPLOMA: graduate diploma, postgraduate diploma, diploma.
- CERTIFICATE: graduate certificate or certificate.
- Keep MBA as programName="MBA" when it identifies the requested program and
  also include degreeTypes=["MASTER"].
- Never infer a degree type when the wording does not support one.

Program, faculty, and department extraction:
- programName is the requested named program or common program label, such as
  "MBA", "Computer Science", "Master of Computer Science", or
  "M.S. Computer Science". Preserve the user's concise label; do not invent a
  university-specific official degree title because titles differ by university.
- Add a concise topicKeywords value when it improves subject matching, for
  example "computer science" for Computer Science master's wording.
- faculty identifies a faculty, school, or equivalent academic unit. Extract it
  when the user explicitly asks about a faculty/school or names one. Do not set
  faculty merely because a program belongs to a faculty.
- department identifies an explicit department request. For "Computer Science
  department", set department="Computer Science"; do not also infer a faculty.
- Prefer exact official text when the user supplied it. For a well-known,
  unambiguous university-specific shorthand, use the verified catalog text:
  - AUB engineering -> "Maroun Semaan Faculty of Engineering and Architecture"
  - AUB business -> "Suliman S. Olayan School of Business"
  - LAU engineering -> "School of Engineering"
  - LAU business -> "Adnan Kassar School of Business"
  - BAU engineering -> "Engineering"
  - BAU business -> "Business Administration"
- Without a university, do not guess which university's official faculty name
  applies. Preserve a clear generic faculty label only when explicitly stated;
  otherwise use topicKeywords or request clarification when required.

City extraction:
- Set filters.city only for an explicit geographic city, such as Beirut,
  Byblos, or Dubai.
- Never treat an academic subject, degree, program, university, faculty, or
  country as a city. In particular, "in Computer Science", "in Master's",
  "in Artificial Intelligence", and "in Computer Science at LAU" have
  city=null. "in Lebanon" is a country constraint, not a city.

Tuition scope interpretation:

- Never infer tuition.scopeLevel from resource, programName, faculty,
  department, university, degreeTypes, or topicKeywords.

- Set scopeLevel only when the user explicitly requests program-level,

  must be null.
