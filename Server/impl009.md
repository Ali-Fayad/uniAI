TASK CODE: AI_CHAT_009_AUTOMATIC_CHAT_TITLES

Recommended Model:
GPT-5.5

Objective

Implement TODO-009 — Automatic Chat Titles.

Replace the current first-message substring title logic with an AI-generated title similar to ChatGPT.

The title must:

- be generated exactly once
- be based ONLY on the first user message
- never use conversation history
- never use conversation memory
- never use retrieval context
- never use assistant responses
- never regenerate automatically
- never block the chat response

The implementation must remain architecture-consistent with the existing AI pipeline.

--------------------------------------------------
Current Architecture
--------------------------------------------------

Current flow:

User Message
→ ChatApplicationService
→ Interpretation
→ Retrieval
→ Ranking
→ Compression
→ Budget
→ AI Response
→ Persist Assistant Message

Currently the chat title is generated by truncating the first user message.

Replace this behavior.

--------------------------------------------------
Target Flow
--------------------------------------------------

First successful chat turn

Persist user message
→ AI response generated
→ assistant message persisted
→ transaction commits
→ AFTER COMMIT
    if this is still the first user turn
        generate AI title
        validate title
        persist title
→ finish

The title generation must be best-effort.

Failure must never affect:

- sendMessage()
- message persistence
- retrieval
- interpretation
- conversation memory
- provider flow

--------------------------------------------------
Architecture
--------------------------------------------------

Create a dedicated application component.

Suggested name:

ChatTitleGenerationManager

Responsibilities:

- determine whether generation should run
- build title request
- invoke AI
- validate output
- persist title

Do NOT place all title logic inside ChatApplicationService.

ChatApplicationService should only trigger the manager after commit.

Provider
--------------------------------------------------

Reuse the existing AI infrastructure.

Reuse:

- AiServicePort
- AiRequest
- AiResponse
- provider abstraction
- prompt loading pattern

Do NOT introduce:

- new provider abstraction
- new provider registry
- provider fallback
- queue infrastructure

--------------------------------------------------
Dedicated Prompt
--------------------------------------------------

Create:

Server/src/main/resources/prompts/chat-title-generator-prompt.txt

The prompt must instruct the AI:

- Generate one concise title.
- Use ONLY the supplied first user message.
- Never answer the question.
- Never use outside knowledge.
- Never use history.
- Never use memory.
- Never use retrieval.
- Never use assistant responses.
- Return plain text only.
- No Markdown.
- No quotes.
- No explanations.
- 2–6 words preferred.
- Maximum approximately 60 characters.
- Preserve the user's language.

Create a dedicated prompt provider similar to the existing prompt providers.

--------------------------------------------------
Request Content
--------------------------------------------------

System Prompt

First User Message

Nothing else.

Specifically do NOT send:

conversation history

conversation memory

retrieved context

assistant response

GraduateKnowledgeQuery

sources

provider metadata

Budget

- title prompt
- first user message
- request overhead
- output reserve

Do NOT include:

history

Reject impossible requests before provider invocation.

--------------------------------------------------
Output Validation
--------------------------------------------------

Treat provider output as untrusted.

Validate:

- not blank
- single line
- plain text
- no Markdown
- no code block
- no explanation
- trim whitespace
- remove surrounding quotes
- remove trailing punctuation
- normalize repeated spaces
- maximum 60 characters

Reject invalid output.

If rejected:

leave title unset.

Do NOT persist a fallback substring.

--------------------------------------------------
Execution Timing
--------------------------------------------------

Generate AFTER COMMIT.

Never perform the remote AI request inside the main transaction.

Never hold database locks during provider invocation.

Persist title in a short independent transaction.

--------------------------------------------------
Generation Rules
--------------------------------------------------

Generate only once.

Never regenerate.

Use only the FIRST user message.

Do NOT use chat state from later messages.

Guard persistence.

Persist only if:

chat.title is still null

If another concurrent update already stored a title:

silently skip.

provider unavailable

timeout

budget rejection

blank response

invalid output

validation failure

persistence conflict

do NOT retry

do NOT fail sendMessage()

do NOT persist placeholder title

leave title null
